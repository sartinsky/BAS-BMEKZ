Призначене для настройки синхронізації даних і зберігання існуючих установок синхронізації даних.

Список Настройок синхронізації даних можна знайти на Робочому столі програми, або відкрити в розділі Адміністрування - Синхронізація даних по відповідній команді.

У вікні виводиться:

  • Програма - найменування програми, з якої налаштована дана синхронізація;
  • Префікс інформаційної бази-кореспондента;
  • Стан синхронізації даних;

  • Інформація за датами виконаних відправок і отримань, для наочності неуспішні відправки та отримання даних відзначаються червоним значком . Значком позначаються попередження. Відображається також стан синхронізації (час, що минув з останньої спроби синхронізації в хвилинах, Зараз, Ніколи):

    • Дані відправлені - відображається час останньої відправки даних;

    • Дані отримані - відображається час останнього отримання даних.
  • - синхронізація даних виконується за розкладом. У цій колонці проставляється аналогічний значок для синхронізацій, які виконуються автоматично за заданим сценарієм;
  • У нижній частині списку виводиться Префікс цієї інформаційної бази.

Стан синхронізації даних

  • У списку у відповідній колонці відображаються Стани синхронізації даних:
    • Минулий запуск - синхронізація успішно працює, при цьому вказується дата запуску синхронізації даних;
    • Не запускалася - синхронізація успішно, але обмін даними ще не проводився. У цьому випадку натисніть кнопку Запустити синхронізацію;
    • Настройка не завершена - настройка синхронізації була перервана. У цьому випадку можна продовжити настройку, натиснувши кнопку Настроїти.
    • Настройка не завершена, отримані дані для зіставлення - в такому стані настройка синхронізації даних між різними програмами відображається в програмі-приймачі в разі прямого підключення і тоді необхідно продовжувати настройку;
    • Отримані дані для зіставлення - для аналізу та зіставлення отриманих даних натисніть кнопку Запустити синхронізацію.

Настройка синхронізації даних

  • Для установки нової синхронізації даних з іншою програмою виберіть відповідну команду з підменю Нова синхронізація даних. Відкриється настройка синхронізації даних.

Перегляд інформації про настройку

  • Для того щоб отримати інформацію про особливості ведення обліку в програмах, між якими нстроєна синхронізація даних і отримати рекомендації з настройки синхронізації даних, натисніть Ще - Докладніше про синхронізації даних з цією програмою.

Синхронізація даних

  • Виберіть потрібну настройку, натисніть Запустити синхронізацію, для того щоб почати синхронізацію даних.
  • Для синхронізації даних між двома програмами запускається зіставлення однакових даних. Така процедура корисна в тих випадках, коли в обох програмах з якихось причин були незалежно один від одного введені одні і ті ж дані. Для зниження ймовірності таких ситуацій можна настроїти часту синхронізацію даних за розкладом.

Зміна синхронізації даних

  • Для того щоб змінити або продовжити настройку синхронізації даних, натисніть кнопку Настроїти.
  • Також можна скористатися подвійним натисненням кнопки миші.

Перегляд складу даних, що відправляються

  • Натисніть кнопку , щоб переглянути склад даних, що надсилаються.

Вивантаження даних для зіставлення

  • Виконайте команду меню Ще - Вивантаження даних для зіставлення.

Настройка синхронізації за розкладом

  • З допомогою команди Ще - Сценарії синхронізації даних можна Настроїти синхронізацію даних за заданим розкладом. У цьому випадку в клієнт-серверному режимі роботи синхронізація даних буде виконуватися в тлі, з допомогою регламентного завдання Синхронізація даних, а у файловому режимі додатково буде потрібно запустити окремий сеанс для виконання регламентних завдань.
  • Також команду можна знайти в контекстному меню по правій кнопці миші.

Перегляд попереджень

  • Пройдіть за посиланням Попередження (в дужках відображається кількість попереджень), для того щоб детальніше ознайомитися з попередженнями при синхронізації даних.

Перегляд журналу реєстрації

  • За допомогою команд меню Ще - Події надсилання або Події отримання даних можна докладно дізнатися, як проходила синхронізація даних з журналу реєстрації програми, відфільтрованого щодо подій синхронізації, а також переглянути кожну подію.

Заміна правил для синхронізації з зовнішнього файлу

  • Для заміни типових правил конвертації об'єктів та правил реєстрації об'єктів правилами з зовнішнього файлу необхідно:
    • Відкрити синхронізацію даних, для якої потрібно змінити правила.
    • Виконати команду Ще - Завантажити правила.
    • У вікні правил конвертації або реєстрації об'єктів встановити значення перемикача в положення Завантажити з файлу на комп'ютері та по кнопці Завантажити вибрати файл правил на комп'ютері або в мережній папці.
    • Звернути увагу, що завантажені коректні правила, які підходять для використовуваних версій програми-джерела і програми-приймача.
    • Натиснути на кнопку Записати і закрити. Завантажені правила будуть застосовані негайно.

Вилучення настройки синхронізації

  • Виділіть настройку, натисніть Ще - Вилучити настройку синхронізації.
  • Увага! Настройка буде вилучена безповоротно.

Дії при відновленні інформаційної бази з резервної копії

  • При відновленні інформаційної бази з резервної копії рекомендується:
    • Тимчасово відключити автоматичне виконання синхронізації даних (зняти прапорець Автоматично за розкладом у всіх сценаріях синхронізації даних);
    • Скасувати надсилання даних, приготованих для інших програм (для кожної настройки синхронізації даних виконати команду Ще – Склад даних, що відправляються, у вікні Реєстрація змін в дереві об'єктів відзначити корінь і виконати команду Скасувати реєстрацію - Всіх об'єктів вибраних типів);
    • Виконати відправку даних, введених в інших програмах з моменту виконання резервної копії цієї інформаційної бази, з інших програм у цю інформаційну базу;
    • Відновити автоматичне виконання синхронізації даних.
  • При втраті значної частини даних рекомендується вилучити всі настройки синхронізації даних з іншими програмами і виконати настройку синхронізації заново. В іншому випадку, може відбутися втрата даних в програмах, з якими настроєна синхронізація, за відправлення в них застарілих даних з відновленої інформаційної бази.
