Настройка параметров учета

Форма предназначена для установки параметров ведения учета, которые являются общими для всех организаций информационной базы.

Остальные параметры учета сгруппированы по закладкам:

  • Банк и касса,
  • Расчеты с контрагентами,
  • Розничная торговля,
  • Запасы,
  • Комиссионная торговля,
  • Производство,
  • Сотрудники и зарплата.

Банк и касса

Если хотя бы одна организация предприятия ведет расчеты в иностранной валюте, в настройках параметров учета нужно установить флажок Ведется учета расчетов в валюте. Выбранный вариант влияет на интерфейс программы.
Из формы по ссылке можно перейти к справочнику валют.

Настраивается аналитический учет на счете 30. Учет на счете 30 по статьям движения денежных средств ведется всегда. Дополнительно установкой соответствующего флажка можно включить ведение аналитического учета по обособленным подразделениям. Это значит, что головной офис и обособленные подразделения ведут отдельные кассовые книги и раздельную нумерацию кассовых ордеров

Важно!
Учет по обособленным подразделениям необходим также для выписки обособленными подразделениями налоговых накладных. В случае, если хотя бы бы одна из организаций предприятия имеет филиалы (обособленные подразделения), самостоятельно выписывающие налоговые накладные, учет ДДС по обособленным подразделениям должен быть включен

Аналитический учет на счете 31 по банковским счетам и статьям движения денежных средств ведется всегда и не настраивается в форме.

Расчеты с контрагентами

Настраивается аналитический учет по счетам взаиморасчетов с контрагентами: 36, 63, 371, 681, 377, 685. Учет по контрагентам и договорам с ними ведется всегда. Дополнительно можно включить учет по документам расчетов. Ведение учета по документам расчетов позволяет контролировать своевременность оплат и отгрузок, а также определять сумму 1-го события с точностью до сделки.

Если в информационной базе ведется учет по документам расчета, то порядок такого учета уточняется в каждом договоре с контрагентом:

  • Если в договоре включен учет по документам расчетов, то документ расчетов указывается вручную в документах, отражающих расчеты с контрагентами.
  • Если в договоре отключен учет по документам расчетов, то документ расчетов определяется автоматически при проведении документов.

 

Настраиваются срок, по истечении которого долг покупателя и долг продавца считается просроченным, соответственно в полях Срок оплаты покупателей и Срок оплаты поставщикам. Срок оплаты учитывается при формировании отчетов для руководителя. 

Розничная торговля

Если хотя бы в одной из организаций ведется розничная торговля, флажок Ведется розничная торговля должен быть установлен.

Настраивается аналитический учет по счету 2822 "Товары в розничной торговле (в НТТ по продажной стоимости)". Этот счет предназначен для учета товаров в рознице по продажным ценам (с использованием счета 285 "Торговая наценка") для складов, являющихся неавтоматизированными торговыми точками (НТТ).

Учет на счете 2822 всегда ведется по складам (торговым точкам). Дополнительно установкой соответствующих флажков можно включить ведение аналитики по номенклатуре или по ставкам НДС.

Примечание
По счету 2822 в разрезе номенклатуры можно анализировать только дебетовые обороты. Остатки в разрезе номенклатуры не хранятся, учет продаж в разрезе номенклатуры не ведется

Запасы

Настраиваются параметры учета материалов, товаров, готовой продукции и других видов товарно-материальных ценностей.

Аналитический учет запасов на счетах учета всегда ведется по номенклатурным позициям (наименованиям товаров, материалов, продукции). Кроме того, дополнительно может быть установлен аналитический учет запасов по партиям и по складам.
Если в учетной политике хотя бы одной из организаций задан способ оценки стоимости материально-производственных запасов ФИФО, то необходимо установить флажок Ведется учет по партиям (документам поступления).

Необходимо выбрать вариант ведения складского учета.

  • Если не ведется учет запасов по складам, то необходимо снять флажок соответствующий субконто По складам (местам хранения).
  • Если складской учет ведется, то можно вести только количественный учет. Для этого необходимо при установленном флажке субконто По складам (местам хранения) выбрать вариант По количеству. Это позволит контролировать наличие запасов на разных складах в количественном выражении, а цена списания запасов будет определяться путем деления общей стоимости данной номенклатурной позиции на всех складах на общее количество.
  • Предусмотрена возможность вести количественный и суммовой складской учет. Для этого необходимо при установленном флажке субконто По складам (местам хранения) выбрать вариант По количеству и сумме. В этом случае цена списания запасов будет определяться по каждому складу отдельно.

При проведении документов выбытия запасов, контролируется их наличие на момент выбытия. При их отсутствии проведение документа отменяется и выдается сообщение об ошибке. Иногда такой контроль не нужен, например, если сначала в информационную базу вводится информация о списании запасов и только потом о поступлении. В этом случае необходимо установить флажок Разрешается списание запасов при отсутствии остатков по данным учета.

Если хотя бы в одной организации ведется учет оборота возвратной тары, то необходимо установить флажок Ведется учет возвратной тары. Тогда в документах движения запасов добавляется закладка "Тара", на которой указывается информация о поступлении и списании возвратной тары.

Также на закладке устанавливается режим вывода колонок «Код» или «Артикул» при формировании печатных форм.

Комиссионная торговля

Если хотя бы в одной из организаций предполагается деятельность связанная с комиссионной торговлей, должен быть установлен флажок Ведется деятельность по договорам комиссии на продажу. Установка флажка изменяет интерфейс программы: в меню появляется отчет комиссионера и отчет комитенту, в договорах контрагента появляются реквизиты способ расчета и % комиссионного вознаграждения, в документе Перемещение товаров - табличная часть "Товары на комиссии".

Производство

Если хотя бы в одной из организаций ведется производственная деятельность, то необходимо установить флажок Ведется производственная деятельность,  что позволяет настраивать тип плановых цен, а также настраивать учет производственной деятельности в учетной политике организаций.

Тип плановых цен определяет плановую себестоимость, по которой в момент производства оцениваются продукция (услуги) в документах Отчет производства за смену и Акт об оказании производственных услуг.

Сотрудники и зарплата

Организация может вести учет расчетов с персоналом по заработной плате и кадровый учет в текущей информационной базе или во внешней программе. Выбор варианта учета расчетов необходимо указать в настройках параметров учета. Выбранный вариант влияет на интерфейс программы и набор документов, которые используются при отражении расчетов с персоналом.

  • Если хотя бы одной из организаций необходимо вести учет по заработной плате по каждому сотруднику, то необходимо установить переключатель По каждому работнику. В этом случае к счету 70 "Расчеты с персоналом по оплате труда" устанавливается субконто "Работники организаций".

  • Если все организации планируют вносить данные по заработной плате в информационную базу сводными проводками по всем работникам, то необходимо установить переключатель Сводно по всем работникам.
