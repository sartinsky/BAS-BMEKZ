Призначена для настройки додаткових реквізитів (відомостей).

Відкривається по команді Додати або Створити в списку Додаткові реквізити (відомості).

Призначення нового додаткового реквізиту (відомості) довіднику або документу

  • В залежності від настройки може бути доступна одна з команд введення нових реквізитів (відомостей):
    • Виберіть відповідний цьому довіднику або документу набір, натисніть Створити. За замовчуванням додатковий реквізит (відомість) може бути доданий тільки в один набір.
    • Виберіть відповідний цьому довіднику або документу набір, натисніть Додати - Новий (Нове) - аналогічно команді Створити. За допомогою цієї команди додатковий реквізит (відомість) може бути доданий тільки один набір.

Заповніть необхідні поля:

  • Напишіть коротке Найменування властивості, яке буде зазначатися в вікнах об'єктів програми.
  • Для того щоб додатковий реквізит при заповненні довідників або документів не залишався порожнім, увімкніть прапорець Заповнювати обов'язково. Для додаткових відомостей прапорець не передбачений.
  • Заповніть поля на вкладці Головне:
    • Виберіть зі списку за допомогою прапорця Тип значення додаткового реквізиту (відомості);
    • Для того щоб уточнити формат додаткових реквізитів типу Число, Дата, Булево, перейдіть в Конструктор форматного рядка за посиланням Формат по умовчанню (для додаткових відомостей не передбачено).
      • Встановіть при необхідності значення в полі Мова (Країна), для того щоб змінити стандартні настройки.
      • Виберіть необхідні параметри для установки форматів.
      • У рядку Приклад для наочності проводяться відповідні цим параметрам зміни.
      • Для підтвердження вибраних параметрів натисніть ОК.
      • Після уточнення формату посилання змінює значення на Формат встановлений.
      • Детальніше про форматах див. нижче.
    • При необхідності увімкніть прапорець Виводити у вигляді гіперпосилання - для розміщення на картці об'єкта (документа, списка) посилань на зовнішні ресурси або інші об'єкти програми (наприклад, файли). Доступно для рядкових додаткових реквізитів і для посилальних об'єктів.
    • Заповніть поле Спливаюча підказка, для того щоб пояснити, як заповнюється додаткову відомість або реквізит. Вміст поля буде виводитися на екран при наведенні миші на поле додаткового реквізиту (відомості).
    • Тільки для додаткових реквізитів: перейдіть за посиланням в поле Видно, для того щоб налаштувати видимість додаткового реквізиту. Видимість додаткового реквізиту може залежати від деякої умови (наприклад, від заповнення іншого реквізиту). Після того як умову задано, посилання відображає задану умову. По умовчанню відображається значення завжди.
    • Тільки для додаткових реквізитів: перейдіть за посиланням в поле Доступний, для того щоб налаштувати доступність додаткового реквізиту для заповнення. Доступність додаткового реквізиту може залежати від деякої умови (наприклад, від заповнення іншого реквізиту). Після того як умову задано, посилання відображає задану умову. По умовчанню відображається значення завжди.
    • Тільки для додаткових реквізитів: увімкніть прапорець Заповнювати обов'язково, для того щоб додатковий реквізит при заповненні списків або документів не залишався порожнім. Може залежати від деякої умови (наприклад, від заповнення іншого реквізиту). Для того щоб задати умову, перейдіть за посиланням. Посилання по умовчанню приймає значення завжди.
  • Заповніть поле Коментар, для того щоб при підборі додаткового реквізиту (відомості) у набори в нижній частині списку вибору виводилося його опис. Для заповнення перейдіть на вкладку Коментар.

Додавання з іншого набору

  • У програмі передбачена можливість використовувати раніше введені додаткові реквізити та відомості для інших наборів.
  • Найбільш актуальний цей спосіб для додаткових реквізитів і відомостей типу Додаткове значення і Додаткове значення (ієрархія). Якщо проведені відповідні настройки (включений хоча б один з прапорців у розділі Адміністрування - Загальні настройки), стає доступною кнопка Додати.
  • Виберіть набір додаткових реквізитів (відомостей), натисніть кнопку Додати, у випадаючому меню виконайте команду З іншого набору.
  • Відкривається помічник з додавання додаткового реквізиту (відомості).
    • У лівій частині першої сторінки помічника відображається список раніше введених реквізитів (відомостей).
    • У правій частині відображаються значення та інші параметри обраного реквізиту.
  • На першій сторінці помічника виберіть додатковий реквізит (зведення). Зверніть увагу, що створити додатковий реквізит або відомість в даному списку неможливо.
    • Для вибору варіанту додавання додаткового реквізиту (відомості) натисніть кнопку Далі.
    • Також можна додати реквізит з одного набору в інший за допомогою перетягування мишею. Якщо були зроблені відповідні настройки, відкривається друга сторінка помічника Додавання додаткового реквізиту (відомості).
  • На другій сторінці помічника за допомогою перемикача виберіть варіант додавання додаткового реквізиту (відомості):
    • Зробити копію реквізиту - в новому наборі буде створено такий самий додатковий реквізит (відомість).
    • Зробити копію реквізиту за зразком - створюється додатковий реквізит (відомість) зі спільним списком значень. Має бути ввімкнено прапорець Реквізити та відомості зі спільним списком значень у розділі Адміністрування - Загальні настройки.
    • Додати спільний реквізит в набір - зробити додатковий реквізит (зведення) спільним. У цьому випадку спільним стає не список значень, а сам реквізит. Повинен бути включений прапорець Спільні реквізити та відомості в розділі Адміністрування - Загальні настройки.

Копіювання додаткових реквізитів (відомостей)

  • У програмі передбачена можливість копіювання додаткових реквізитів (відомостей).
  • В цьому випадку для набору додаткових реквізитів (відомостей) буде створена копія - самостійний додатковий реквізит (відомість) зі своїм списком значень, ніяк не пов'язаний з оригіналом, за яким був створений.
  • Залежно від настройок проводиться декількома способами:
    • В помічнику додавання додаткових реквізитів і відомостей;
      • Виділіть набір додаткових реквізитів у списку, натисніть Додати - З іншого набору.
      • На першій сторінці помічника в списку вибору реквізитів для копіювання виберіть потрібний реквізит, натисніть Далі.
      • На другій сторінці помічника за допомогою перемикача виберіть Зробити копію реквізиту, натисніть Далі.
      • Заповніть бракуючі поля. Після копіювання реквізиту список значень доступний для редагування.
    • Перетягуванням мишею;
      • Можна перетягнути потрібний додатковий реквізит (відомість) з одного набору в інший за допомогою миші.
      • Якщо у списку є кнопка Додати, то відкривається друга сторінка помічника додавання додаткового реквізиту (відомості).
      • Якщо в списку тільки кнопка Створити, то відкривається картка додаткового реквізиту (відомості), який буде скопійовано. Заповніть бракуючі поля.
    • За допомогою кнопок Копіювати і Вставити.
      • Виділіть додатковий реквізит (відомість), натисніть кнопку Копіювати. Виділіть набір, в який потрібно додати додатковий реквізит (відомість). Натисніть кнопку Вставити. Якщо були зроблені відповідні настройки, то відкривається друга сторінка помічника додавання додаткових реквізитів (відомостей). Якщо обраний додатковий реквізит (відомість) вже входить в який-небудь набір, то пропонується зробити його спільним.
      • Команди Копіювати і Вставити також доступні в контекстному меню по правій кнопці миші або в меню Ще.

Додавання за зразком

  • Для того щоб використовувати цю можливість, необхідно увімкнути прапорець Реквізити та відомості з загальним списком значень у розділі Адміністрування - Загальні настройки.
  • Можна використовувати раніше введені додаткові реквізити та відомості з інших наборів, додавати реквізити із загальним списком значень відразу для декількох однотипних реквізитів (наприклад, Колір двері за зразком реквізиту Колір). Список значень в цьому випадку використовується з реквізиту-зразка. Зручно виконувати централізовану настройку списку значень одразу для декількох однотипних реквізитів.
  • Виберіть набір зі списку.
  • Натисніть Додати - З іншого набору.
  • Буде відкрито список всіх додаткових реквізитів (відомостей), вже існуючих в програмі. Виберіть додатковий реквізит (зведення) зі списку вже існуючих в програмі, натисніть Далі.
  • За допомогою перемикача виберіть спосіб створення елемента Зробити копію реквізиту за зразком, натисніть Далі.
  • Тип і всі значення обраного в якості зразка додаткового реквізиту (відомості) копіюються в новий елемент.
  • Заповніть всі необхідні поля. За замовчуванням реквізит стає загальним з набором, з якого був використаний зразок. Введення нових значень в цьому випадку неможливе. Програма виводить нагадування, що список значень є загальним з іншим реквізитом.
  • Після збереження додаткового реквізиту (відомості) за допомогою посилання можна переглянути реквізит-зразок і додати значення, якщо необхідно.
  • Реквізит, створений за зразком, який має загальний список значень з іншим реквізитом, можна зробити окремим реквізитом.
    • Для цього потрібно натиснути кнопку Змінити у вікні Змінити настройку додаткового реквізиту (відомості) вибрати перемикач Реквізит зі своїм списком значень і записати зміни. Список значень буде скопійований з заміною посилань в документах. Після цього стане доступним введення списку значень.
    • Увага: дану перенастройку для вже діючих реквізитів (використаних для вставки в документи) слід застосовувати з обережністю. Перенастроювання реквізиту є незворотною, може виконуватися автоматично, але займає багато часу і може закінчитися некоректно. Можна замінити значення у вже існуючих документах вручну, очистивши старі значення і додавши нові (рекомендується).

Додавання загальних додаткових реквізитів (відомостей)

  • Спільний реквізит входить в кілька наборів. Цей варіант підходить для тих випадків, коли спільним повинен бути не тільки список значень, але і сам реквізит. У списках і звітах за елементами різних довідників і документів спільні реквізити групуються і виводяться як один реквізит в одній колонці або полі. Наприклад, з допомогою спільного реквізиту Колір можна відібрати всю жовту одяг і взуття.
  • Для того щоб використовувати цю можливість, необхідно включити прапорець Спільні реквізити та відомості в розділі Адміністрування - Загальні настройки.
  • Для того щоб додати в набір загальний додатковий реквізит (зведення), виберіть потрібний набір, натисніть Додати - З іншого набору.
  • Виберіть із запропонованого списку вже існуючий загальний додатковий реквізит (відомість). Натисніть кнопку Далі.
  • Відкривається друга сторінка помічника. Виберіть за допомогою перемикача варіант додавання Зробити реквізит загальним і додати в набір, натисніть Далі. Відкривається загальний додатковий реквізит.
  • Також можна додати загальний додатковий реквізит (відомість) за допомогою перетягування мишею або кнопок Копіювати і Вставити.
  • Для того щоб додати загальний реквізит в кілька наборів, повторіть ці дії для кожного набору.
  • В нижній частині вікна загального додаткового реквізиту (відомості) при розміщенні в одному або кількох наборах з'являється посилання Спілний реквізит (Спільна відомість) входить в набір з переліком наборів.
  • За допомогою кнопки Змінити (знаходиться поруч з посиланням) можна спыльний реквізит зробити окремим реквізитом з власним списком значень.
    • У вікні Змінити настройку додаткового реквізиту (відомості), натисніть перемикач Реквізит зі своїм списком значень і записати зміни. Список значень буде скопійований з заміною посилань в документах.
    • Увага! Перенастройка додаткового реквізиту (відомості) необоротна і може зайняти тривалий час, залежно від числа документів, в яких заповнений цей реквізит. Замість цього можна додати новий реквізит зі своїм списком значень і виконати вручну поступовий перехід від використання цього реквізиту до нового (очистити значення цього реквізиту і заповнити значення нового).

Типи значень додаткових реквізитів і відомостей

Тип значення додаткових реквізитів і відомостей змінюється виходячи з логіки використання. Можливі такі типи додаткового реквізиту або відомості:

  • Рядок - Визначте Довжину рядка виберіть параметр Фіксована або Змінна (якщо введена рядок вийде менше передбачуваної довжини, то кількість знаків, відведений під це значення буде автоматично зменшено до фактичної). Максимальна довжина рядка в цьому випадку 1024 символів. Також можна включити прапорець Необмежена, якщо планується вводити великий обсяг текстової інформації.
    • Для додаткового реквізиту типу Строка у програмі пропонується можливість зробити поле багаторядковим. Для цього увімкніть прапорець Багаторядкове поле введення і введіть кількість рядків.
  • Число - виберіть Довжину числа і його Точність. По умовчанню програма пропонує довжину числа 15 і точність 3. Залежно від даних, що вводяться, ці цифри можуть змінюватися. Необхідно враховувати, що в кількість знаків Довжини числа входить кількість знаків цілої частини, і десяткова точка, і кількість знаків дробової частини. При необхідності увімкніть прапорець Невід'ємне.
    • При введенні додаткового реквізиту типу Число передбачена можливість більш точного вибору формату.
      • При введенні значень в поля Довжина і Точність в Конструкторі форматного рядка число буде округлятися до цієї кількості знаків.
      • Можна вибрати Зсув.
      • Можна поміняти Роздільник дробової частини (по умовчанню - кома), визначити Роздільник груп (по умовчанню пробіл), також визначити Группування.
      • Можна вибрати Представлення нуля, наприклад, для того щоб нагадати про необхідність введення значення в це поле.
      • Також можна вказати Представлення негативних чисел.
      • При необхідності увімкніть прапорець Виводити лідируючі нулі. У цьому випадку при підгонці під вказану Довжину числа, якщо число менше, будуть проставлені нулі в розрядах.
    • Увага! Деякі настройки формату (наприклад, представлення порожнього числа, довга дата і деякі інші) автоматично не застосовуються в більшості місць і підтримуються тільки там, де розробник це передбачив (наприклад, в друкованих формах).
      • У картках об'єктів вони незастосовні.
  • Дата - виберіть склад дати: Дата, Час або Дата і Час.
    • Для додаткових реквізитів можна задати формат дати, для цього перейдіть за посиланням Формат по умовчанню. Можна вибрати формат dd.mm.yyyy або yyyy-MM-dd, а також Локальний формат дати (наприклад, 22 травня 2013), задати Представлення порожньої дати (в незаповненому полі з'явиться попередження, наприклад, Немає дати).
      • Увага! Деякі настройки формату (наприклад, представлення порожнього числа, довга дата і деякі інші) автоматично не застосовуються в більшості місць і підтримуються тільки там, де розробник це передбачив (наприклад, в друкованих формах).
      • У картках об'єктів вони незастосовні.
  • Складений тип даних - цей тип значення, надає можливість остаточно вибрати тип даних вже при введенні інформації в довідник. Увімкніть прапорець Складений тип даних, потім увімкніть прапорці для типів значень, з яких можна буде вибирати.
    • Наприклад, дату заснування фірми іноді неможливо вказати точно. У цьому випадку можна надати можливість вибору типу даних (Дата або Строка) під час введення. Якщо точна дата відома, то можна буде вибрати тип Дата і ввести, якщо невідома, то можна буде ввести дату у вигляді рядка типу "Початок 19 ст.".
    • При цьому для всіх частин складеного типу даних можна зробити їх звичайні настройки, наприклад, настроїти формат дати і визначити параметри рядка.
    • При введенні значень додаткових реквізитів (відомостей) складеного типу даних спочатку пропонується вибрати тип даних.
  • Булево - зазвичай це значення типу Істина і Хибність, Так чи Ні.
    • Для додаткових реквізитів у Конструкторі форматного рядка можна встановити інший формат цих значень. Можна написати Представлення значення булево Хибність і булево Істина. У цьому випадку при введенні замість звичайного Так чи Ні для вибору будуть запропоновані значення, введені підчас настройки додаткового реквізиту.
      • Увага! У списках і звітах відбір для таких додаткових реквізитів з умовою Дорівнює і Не дорівнює не підтримується.
  • Додаткове значення - цей тип даних часто використовується, при цьому введення інформації зводиться до вибору значень зі списку. Пропонується програмою по умовчанню. Застосовується як звичайний список значень, так і згрупований. З допомогою додаткових реквізитів (відомостей) типу Додаткове значення можна описувати різні властивості об'єктів програми.
  • Додаткове значення (ієрархія) - така структура списків також широко використовується, її називають ієрархією елементів. Елементи в такому списку мають однакові властивості, але можуть мати підлеглі елементи. При цьому ієрархія може бути досить складною.
  • Користувач - у програмі передбачена можливість використання значень списку Користувачі (та інших списків) в якості додаткового реквізиту (відомості).

Особливості введення додаткових відомостей типу Додаткове значення або Додаткове значення (ієрархія)

  • Крім звичайних полів доступні:
    • Значення додатково характеризуються ваговим коефіцієнтом - увімкніть прапорець, для того щоб використовувати цю можливість. Поле потім можна використовувати для аналізу в звітах. Логіку використання цього поля необхідно продумувати самостійно. Поле не є обов'язковим.
    • Заголовок форми значення - введіть заголовок вікон введення елементів (вікон введення елементів з ієрархією).
    • Заголовок форми вибору значення - введіть заголовок вікон вибору значень (вікон вибору значень з ієрархією). Застосування індивідуальних заголовків для вікон введення додаткових реквізитів і відомостей і вибору значень дозволяє зробити процес введення більш зручним і зрозумілим.
  • Після вибору типу даних і заповнення поля Найменування натисніть кнопку Записати. Після цього стає доступним для роботи список значень. Для введення даних натисніть кнопку Створити. Введіть необхідні поля.
  • Якщо дані не були записані перед виконанням команди Створити, то програма видає попередження. Для переходу до введення значень натисніть кнопку Записати.

Зміна формату додаткових реквізитів і відомостей

  • Будь-які зміни типів значень і формату додаткових реквізитів і відомостей після збереження неможливі. Для цього доведеться спочатку вилучити створені додаткові реквізити і відомості, потім вже заново створити додаткові реквізити і відомості з новими типами значень.
