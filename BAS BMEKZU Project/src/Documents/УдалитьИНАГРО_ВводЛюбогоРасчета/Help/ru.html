Документ "Ввод любого расчета"

Документ "Ввод любого расчета" предназначен для ввода любого начисления или удержания. В шапке документа указываются необходимые реквизиты, в частности вид расчета и месяц начисления.

Возможны два режима работы документа: "Ввод начислений" и "Ввод удержаний". Режим работы определяется автоматически, исходя из того, из какого плана видов расчета выбран вид расчета. По умолчанию устанавливается режим "Начисление"; при необходимости его изменить, следует с помощью кнопки очистки поля "Вид расчета" очистить данное поле, после чего выбрать нужный план видов расчета и затем вид расчета.

При заполнении табличной части "Начисления" ("Удержания") можно воспользоваться кнопкой "Заполнить", в результате чего табличная часть заполнится сотрудниками данного подразделения или всей организации (в случае, если поле "Подразделение" не заполнено). Если для вводимого расчета необходимы дополнительные параметры (например величина процента для премии), их также можно ввести для всех сотрудников в табличной части с помощью кнопки "Заполнить".

На закладке "Дополнительно" необходимо указать счет и аналитику затрат. Данный счет и аналитика будет использована в проводках для всех сотрудников, которые указаны в табличной части. При необходимости указать для некоторых сотрудников другой счет и аналитику затрат, это можно сделать непосредственно в табличной части для этого сотрудника, при этом приоритетом пользуется счет и аналитика,  которая введенна в табличной части.

При необходимости на закладке дополнительно можно установить другой, отличный от введенного по умолчанию, базовый период. При данном действии будут соответствующим образом изменены все строки в табличной части "Начисления".

Расчет записей документа выполняется при нажатии на кнопку "Рассчитать". При этом, если выбранный вид расчета имеет нужду в расчетной базе, возможны два варианта последующего хода событий:

  • ни по одному из сотрудников, указанных в табличной части, не была найдена расчетная база. В этом случает система выдаст соответствующее предупреждение и предложит записать документ не рассчитывая. В конце месяца такие "нулевые" записи будут рассчитаны документом "Начисление зарплаты работникам организаций";

  • по некоторым сотрудникам, из указанных в табличной части, не была найдена расчетная база. В этом случает система выдаст соответствующее предупреждение и предложит очистить документ от таких сотрудников. Если согласится с данным предложением, документ очистится от сотрудников, по которым не была найдена расчетная база, и рассчитает записи по тем сотрудникам, по которым база была найдена, после чего можно провести документ. Если же отказаться от данного предложения, документ не будет рассчитывать записи ни по одному сотруднику из числа указанных в табличной части, в этом случае можно действовать также, как в первом случае - провести документ с нулевыми записями.

Важно! При расчете видов расчета, имеющих нужду в базе, необходимо особое внимание обращать на базовый период - именно из этого временного промежутка анализируются начисления по базовым видам расчета.
