Документ "Реєстрація авансів в податковому обліку"

Документ "Реєстрація авансів в податковому обліку" призначений для коригування очікуваного податкового кредиту і податкових зобов'язань по всіх незакритих авансами, для договорів для яких встановлено метод «по оплаті» або «по першій події» з обліку ПДВ, а також закриття "неповних" повернень (див. нижче).

Документ вводиться в кінці податкового періоду, після реєстрації всіх первинних (відвантажувальних, податкових) документів. На початку наступного податкового періоду прийняті коригування будуть компенсовані з метою збереження принципу роботи "по відвантаженню" протягом податкового періоду.

Після автоматичного заповнення користувачеві необхідно визначити параметри податкового обліку для кожного реєстрованого "авансу".

Для видів операцій "Аванс: визначення параметрів ПО", "Аванс (повернення): визначення параметрів ПО" необхідно вибрати ставку ПДВ, податкове призначення, за яким аванс повинен бути зареєстрований. Ці види операцій можуть бути присутніми для договорів як моментом визначення податкової бази «по оплаті», так і «по першій події», у разі якщо оплат зареєстровано більше ніж відвантажень.

Для видів операцій "Відвантаження: скасування реєстрації в ПО", "Відвантаження (повернення): скасування реєстрації в ПО" необхідно вибрати ставку ПДВ, за якою потрібно відмінити реєстрацію очікуваного ПДВ на суму відвантажень, що перевищують оплату. Ці види операцій можуть бути присутніми тільки для договорів з моментом визначення податкової бази «по оплаті», у разі якщо відвантажень зареєстровано більше ніж оплат.

В залежності від того, які методи визначення податкової бази вибрані для конкретного договору, в рядку проставляється ознака "ПДВ" ("Реєстрація в обліку ПДВ"). За цими ознаками можна визначити, чи буде виконуватися коригування (визначення параметрів для авансу або скасування реєстрації відвантаження) за конкретною рядку. Ознака "ПДВ" визначається автоматично при заповненні по залишках чи при виборі договору.

При проведенні документа, за табличній частині "Придбання":

- формуються коректувальні рухи по регістрах "Очікуваний і підтверджений ПДВ Придбань", "Придбання податковий облік";

- бухгалтерські проводки по госпрозрахунковому планом рахунків між рахунком обліку податкового кредиту (вибирається в реквізиті "Рахунок обліку ПДВ") та рахунком непідтвердженого податкового кредиту (вибирається в реквізиті "Рахунок обліку непідтвердженого ПДВ"), на суму коригування очікуваного податкового кредиту;

 

У табличній частині "Продажі":

- формуються коректувальні рухи по регістрах "Очікуваний і підтверджений ПДВ Продажів", "Продажі податковий облік";

- бухгалтерські проводки по госпрозрахунковому планом рахунків між рахунком обліку непідтверджених податкових зобов'язань (вибирається в реквізиті "Рахунок обліку непідтвердженого ПДВ") та рахунком податкових зобов'язань (вибирається в реквізиті "Рахунок обліку ПДВ"), на суму коригування очікуваних податкових зобов'язань;

 

Для всіх рухів на момент початку наступного податкового періоду (місяця) прийняті коригування сторнуються (формуються "зворотні" руху і проводки) з метою збереження принципу роботи "по відвантаженню" для наступного податкового періоду.

 

Ще однією функцією документа є закриття в податкових регістрах "неповних" операцій повернення. Наприклад: покупець повернув відвантажений йому товар, причому ні оплати, ні повернення оплати не було.

Для цієї операції призначена закладка "Повернення".

Після автоматичного заповнення даних на закладці користувач може коригувати суми в колонці «Сума повернення» (позитивна сума означає повернуте відвантаження, негативна - оплату).

Якщо встановлений прапорець "Не передбачається операція, що закриває" - при проведенні будуть сформовані проводки по регістру "Продажу податковий облік" або "Придбання податковий облік" (в залежності від типу контрагента) в припущенні, що, наприклад, за товарами, щол повренені, не передбачається взаємного перерахування грошей (повернення оплати) .

Для формування коректних проводок при закритті неповних повернень важливо, щоб і реєстрація авансів і закриття повернень (відносяться до одного і того ж договору) здійснювалися одним документом.

 

ЗАУВАЖЕННЯ: При проведенні документ "закриває" регістри "Продажі податковий облік" та "Придбання податковий облік", тобто, списує з регістра дані по взаєморозрахунках з контрагентами, пов'язані з закритим ("повним") операціями, тобто за тими операціями, за якими відбулися і відвантаження та оплата (або взаємні відвантаження у разі бартерного договору).
