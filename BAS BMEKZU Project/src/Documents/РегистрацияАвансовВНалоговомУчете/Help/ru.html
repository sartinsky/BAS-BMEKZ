Документ "Регистрация авансов в налоговом учете"

Документ "Регистрация авансов в налоговом учете" предназначен для корректировки ожидаемого налогового кредита и налоговых обязательств по всем незакрытым авансам, для договоров для которых установлен метод «по оплате» либо «по первому событию» по учету НДС, а также закрытие "неполных" возвратов (см. ниже).

Документ вводится в конце налогового периода, после регистрации всех первичных  (отгрузочных, налоговых) документов. В начале следующего налогового периода принятые корректировки будут компенсированы с целью сохранения принципа работы "по отгрузке" в течение налогового периода.

После автоматического заполнения пользователю необходимо определить параметры налогового учета для каждого регистрируемого "аванса".

Для видов операций "Аванс: определение параметров НУ", "Аванс (возврат): определение параметров НУ" необходимо выбрать ставку НДС, налоговое назначение, по которым аванс должен быть зарегистрирован. Эти виды операций могут присутствовать для договоров как моментом определения налоговой базы «по оплате», так и с «по первому событию», в случае если оплат зарегистрировано больше чем отгрузок.

Для видов операций "Отгрузка: отмена регистрации в НУ", "Отгрузка (возврат): отмена регистрации в НУ" необходимо выбрать ставку НДС, по которым нужно отменить регистрацию ожидаемого НДС на сумму отгрузок, превышающих оплату. Эти виды операций могут присутствовать только для договоров с моментом определения налоговой базы «по оплате», в случае если отгрузок зарегистрировано больше чем оплат.

В зависимости от того, какие методы определения налоговой базы выбраны для конкретного договора, в строке проставляется признак "НДС" ("Регистрация по учету НДС"). По этим признакам можно определить, будет ли выполняться корректировка (определение параметров для аванса или отмена регистрации отгрузки) по конкретной строке. Признак "НДС" определяется автоматически при заполнении по остаткам ли при выборе договора.

При проведении документа, по табличной части "Приобретения":

- формируются корректировочные движения по регистрам "Ожидаемый и подтвержденный НДС Приобретений", "Приобретения налоговый учет";

- бухгалтерские проводки по хозрасчетному плану счетов между счетом учета налогового кредита (выбирается в реквизите "Счет учета НДС") и счетом неподтвержденного налогового кредита (выбирается в реквизите "Счет учета неподтвержденного НДС"), на сумму корректировки ожидаемого налогового кредита;

 

По табличной части "Продажи":

- формируются корректировочные движения по  регистрам "Ожидаемый и подтвержденный НДС Продаж", "Продажи налоговый учет";

- бухгалтерские проводки по хозрасчетному плану счетов между счетом учета неподтвержденных налоговых обязательств (выбирается в реквизите "Счет учета неподтвержденного НДС") и счетом налоговых обязательств (выбирается в реквизите "Счет учета НДС"), на сумму корректировки ожидаемых налоговых обязательств;

 

Для всех движений на момент начала следующего налогового периода (месяца) принятые корректировки сторнируются (формируются "обратные" движения и проводки) с целью сохранения принципа работы "по отгрузке" для следующего налогового периода.

 

Еще одной функцией документа является закрытие в налоговых регистрах "неполных" операций возврата. Например: покупатель вернул отгруженный ему товар, причем ни оплаты ни возврата оплаты не было.

Для этой операции предназначена закладка "Возвраты".

После автоматического заполнения данных на закладке пользователь может откорректировать суммы в колонке "Сумма возвращенной отгрузки (оплаты)" (положительная сумма обозначает возвращенную отгрузку, отрицательная - оплату).

Если установлен флажок "Не предполагается закрывающая операция" - при проведении будут сформированы проводки по регистру "Продажи налоговый учет" или "Приобретения налоговый учет" (в зависимости от типа контрагента) в предположении, что, например, по возвращенным товарам не предполагается взаимного перечисления денег (возврата оплаты) .

Для формирования корректных проводок при закрытии неполных возвратов важно, чтобы и регистрация авансов и закрытие возвратов (относящиеся к одному и тому же договору) осуществлялись одним документом.

 

ЗАМЕЧАНИЕ: При проведении документ "закрывает" регистры "Продажи налоговый учет" и "Приобретения налоговый учет", то есть, списывает из регистра данные по взаиморасчетам с контрагентами, относящиеся закрытым("полным") операциям, то есть по тем операциям, по которым произошли и отгрузка и оплата (или взаимные отгрузки в случае бартерного договора).
