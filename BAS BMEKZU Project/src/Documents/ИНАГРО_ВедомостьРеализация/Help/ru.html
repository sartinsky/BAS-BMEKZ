Документ "Ведомость реализация"

Документ "Ведомость реализация" предназначен для реализации одного вида ТМЦ или услуги нескольким контрагентам, пайщикам (в счет пая) или сотрудникам (в счет зарплаты). Допустимы три вида операции:

  • В счет зарплаты;

  • Реализация;

  • В счет пая.

Реализация в счет зарплаты

Для реализации ТМЦ (услуги) сотрудникам в счет зарплаты необходимо заполнить табличную часть документа сотрудниками, которым производится выплата. Для автоматического выполнения этого действия можно воспользоваться командой "Сотрудниками по подразделению" подменю "Заполнить" командной панели табличной части. В результате чего откроется форма выбора подразделения. После выбора конкретного подразделения табличная часть заполнится сотрудниками этого подразделения. Можно также заполнить табличную часть сотрудниками всей организации, для этого достаточно, не производя выбора подразделения, просто закрыть форму выбора.

После этого для каждого сотрудника нужно указать количество реализовываемой номенклатуры или услуги. Если всем сотрудникам предполагается реализовывать одинаковое количество номенклатуры или услуги, это также можно выполнить автоматически с помощью команды "Всем одно количество" подменю "Установить" командной панели табличной части.

Также автоматически можно установить всем сотрудникам, которым производится реализация, одну цену реализовываемой номенклатуры, для этого можно воспользоваться командой "Всем одну цену" подменю "Установить" командной панели табличной части. Также цену реализации для всех сотрудников можно автоматически поменять в поле "Цена", которое появляется, если установить флаг "Указать учетную цену".

Реализация

Для реализации ТМЦ  (услуги) списку контрагентов необходимо заполнить табличную часть документа контрагента, которым предполагается реализовывать номенклатуру. Принцип установки цены номенклатуры и количества аналогичен предыдущему виду операции.

В счет пая

Данный вид операции предназначен для погашения задолженности перед пайщиком после её начисления. Можно одновременно начислить эту задолженность, если на закладке "Подоходный" установить флаг "Начислять подоходный". После установки данного флага становится доступным еще один флаг "Начислять по паям", установив его, можно сделать проводки по начислению в корреспонденции с указанным счетом затрат.

В командной панели табличной части после установки флага "Начислять подоходный" появляется кнопка "Рассчитать НДФЛ", которая позволяет пересчитать сумму НДФЛ.

Характерно для этого вида операции то, что в подменю "Установить" командной панели формы появляется еще две команды:

1) Количество от доли пая - предлагается ввести количество на один пай, после чего количество реализовываемой номенклатуры будет посчитано как произведение доли пая пайщика и введенного пользователем количества на один пай.

2) Количество от площади по кадастру - пользователю предлагается ввести два показателя: площадь по кадастру на один пай и количество на один пай. После ввода этих показателей количество реализовываемой номенклатуры посчитается по формуле:

КоличествоРеализовываемойНоменклатуры = КоличествоНаОдинПай * ПлощадьЗемлиПоКадастру / ПлощадьПоКадаструНаОдинПай.

Для всех трех видов операции характерны некоторые общие принципы заполнения реквизитов документов.

На закладке дополнительно необходимо заполнить все незаполненные реквизиты.

Как уже описывалось выше, если установить флаг "Указать учетную цену" и указать в появившемся поле цену реализации, она будет изменена во всей табличной части. Также во всей табличной части меняется ставка НДС, если ее изменить в шапке документа. При этом пересчитываются суммы НДС в табличной части.
