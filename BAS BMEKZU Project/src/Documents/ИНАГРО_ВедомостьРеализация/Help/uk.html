Документ "Відомість реалізація"

Документ "Відомість реалізація" призначений для реалізації одного виду ТМЦ чи послуги декільком контрагентам, пайщикам (у рахунок паю) чи співробітникам (у рахунок зарплати). Допустимі три види операції:

  • У рахунок зарплати;

  • Реалізація;

  • У рахунок паю.

Реалізація у рахунок зарплати

Для реалізації ТМЦ (послуги) співробітникам у рахунок зарплати необхідно заповнити табличну частину документу співробітниками, яким проводиться виплата. Для автоматичного виконання цієї дії можна скористатися командою "Співробітниками по підрозділу" підменю "Заповнити" командної панелі табличної частини. В результаті чого відкриється форма вибору підрозділу. Після вибору конкретного підрозділу таблична частина заповниться співробітниками цього підрозділу. Можна також заповнити табличну частину співробітниками всієї організації, для цього достатньо, не роблячи вибору підрозділу, просто закрити форму вибору.

Після цього для кожного співробітника потрібно вказати кількість номенклатури чи послуги, що реалізується.  Якщо для всіх співробітників передбачається реалізовувати однакову кількість номенклатури чи послуги, це також можна виконати автоматично за допомогою команди "Всім одна кількість" підменю "Встановити" командної панелі табличної частини.

Також автоматично можна встановити всім співробітникам, яким проводиться реалізація, одну ціну номенклатури, що реалізується. Для цього можна скористатися командою "Всім одну ціну" підменю "Встановити" командної панелі табличної частини. Також ціну реалізації для всіх співробітників можна автоматично змінити в полі "Ціна", яке з'являється, якщо встановити прапорець  "Вказати облікову ціну".

Реалізація

Для реалізації ТМЦ  (послуги) списку контрагентів необхідно заповнити табличну частину документа контрагента, яким передбачається реалізовувати номенклатуру. Принцип встановлення ціни номенклатури і кількості аналогічний попередньому виду операції.

У рахунок паю

Даний вид операції призначений для погашення заборгованості пайщику після її нарахування. Можна одночасно нарахувати цю заборгованість, якщо на закладці "Прибутковий податок" встановити прапорець "Нараховувати на прибуток". Після встановлення даного прапорця стає доступним ще один прапорець "Нараховувати по паях", встановивши його, можна зробити проводки по нарахуванню в кореспонденції з вказаним рахунком витрат.

Після встановлення прапорця "Нараховувати на прибуток" в командній панелі табличної частини з'явиться кнопка "Розрахувати ПДФО", котра дозволяє перерахувати суму ПДФО.

Характерно для цього виду операції те, що в підменю "Встановити"командної панелі форми з'являються ще дві команди:

1) Кількість від долі паю - пропонується ввести кількість на один пай, після чого кількість номенклатури, що реалізується, буде пораховано як добуток долі паю пайщика на  введену користувачем кількість на один пай.

2) Кількість від площі по кадастру - користувачу запропоновано ввести два показники: площу по кадастру на один пай та кількість на один пай. Після вводу цих показників кількість номенклатури, що реалізується, розрахується за формулою:

КількістьРеалізовуємоїНоменклатури = КількістьНаОдинПай * ПлощаЗемліПоКадастру / ПлощаПоКадаструНаОдинПай.

Для всіх трьох видів операції характерні деякі загальні принципи заповнення реквізитів документів.

На закладці Додатково необхідно заповнити всі незаповнені реквізити.

Якщо встановити прапорець "Вказати облікову ціну" та вказати у полі, що з'явилося, ціну реалізації, вона будет змінена у всій табличній частині. Також  у всій табличній частині зміниться ставка ПДВ, якщо її змінити в шапці документу. При цьому перерахуються суми ПДВ у табличній частині.
