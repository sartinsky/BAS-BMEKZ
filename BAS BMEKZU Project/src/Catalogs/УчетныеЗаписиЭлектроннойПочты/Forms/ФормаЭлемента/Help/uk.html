Опис властивостей облікового запису електронної пошти.

При введенні облікового запису більшість параметрів проставляється програмою по умовчанню в залежності від настройок поштових серверів облікового запису електронної пошти.

Для системних облікових записів рекомендується запам'ятати пароль, для того щоб програма могла розсилати повідомлення від такої облікового запису без участі користувача. Проте, рішення про зберігання паролів в інформаційній базі приймається в кожному випадку індивідуально.

Основні відомості

  • Введіть Найменування облікового запису. Необхідно сформулювати найменування так, щоб воно коротко характеризувало обліковий запис, відображало призначення облікового запису або його власника, наприклад, "Служба техпідтримки". Саме найменування буде з'являтися в різних списках.
  • Введіть Адресу електронної пошти облікового запису у форматі User@Server.net.
  • Вкажіть Ім'я відправника листів - ім'я власника облікового запису, яке буде вказуватися в кореспонденції, при цьому воно може відрізнятися від імені користувача в програмі, наприклад, користувач "Адміністратор", а ім'я його "Іван Іванович".
  • Вкажіть, як Використовувати обліковий запис:
    • Увімкніть прапорець Для надсилання листів;
    • Увімкніть прапорець Для отримання листів.
  • Вкажіть, Для кого обліковий запис:
    • Для всіх;
    • Тільки для (вкажіть ім'я користувача, вибравши його зі списку).

Відправка листів (настройка сервера вихідної пошти)

  • У полі Сервер вкажіть адресу SMTP-сервера, наприклад, "smtp.Server.net".
    • Вкажіть Порт комп'ютера для надсилання поштових повідомлень, наприклад, "465".
    • За допомогою перемикача вкажіть, коли буде проводитися Шифрування:
      • Авто;
      • Завжди (SSL).
  • За необхідності увімкніть прапорець Потрібна авторизація, за допомогою перемикача виберіть, на якому сервері:
    • на сервері вихідної пошти (SMTP):
      • Вкажіть Ім'я користувача облікового запису у форматі User@Server.net він може відрізнятися від основного, в такому разі він буде призначений тільки для надсилання листів.
      • При необхідності введіть Пароль облікового запису (якщо він не введений, програма запитує його при надсиланні листів).
    • на сервері вхідної пошти (POP).
  • При необхідності увімкніть прапорець Надсилати приховані копії листів на цю адресу.

Отримання листів (настройка сервера вхідної пошти)

  • Вкажіть Ім'я користувача облікового запису у форматі User@Server.net він може відрізнятися від основного, в такому разі він буде призначений тільки для отримання листів.
  • При необхідності введіть Пароль облікового запису (якщо він не введений, програма запитує його для отримання листів).
  • Виберіть Протокол сервера вхідної пошти: POP або IMAP.
  • У полі Сервер вкажіть адресу POP3 сервера, наприклад, "imap.Server.net".
  • Вкажіть Порт комп'ютера для отримання вхідної пошти, наприклад, 993.
  • Виберіть, коли проводити Шифрування:
    • Авто;
    • Завжди (SSL).
  • Якщо вибрано протокол сервера вхідної пошти POP, можна змінити настройки зберігання електронної пошти на сервері:
    • Увімкніть прапорець Залишати копії повідомлень на сервері, для того щоб програма не вилучила листи в папці "Вхідні" поштової скриньки електронної пошти. По умовчанню прапорець відключений, в цьому випадку всі листи після отримання будуть вилучені з сервера. Після цього стає доступним прапорець:
      • Визначте строк зберігання листів на поштовому сервері за допомогою прапорця Вилучати листи з сервера - наприклад, через "10 днів".

Інші настройки

  • Вкажіть Тривалість очікування сервера - наприклад, "30 сек".

Перевірка настройок

  • Натисніть Перевірити настройки для перевірки введених параметрів облікового запису.

  • Програма надсилає стандартне поштове повідомлення і перевіряє можливість з'єднання з сервером вхідних поштових повідомлень (якщо обліковий запис використовується для отримання пошти). Також можна скористатися однойменною командою меню Ще.
