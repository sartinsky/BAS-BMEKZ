У даній панелі здійснюється настройка синхронізації даних з іншими програмами та організація спільної роботи в розподіленій інформаційній базі.

  • Синхронізація даних - увімкніть прапорець, для того щоб стали доступними розділ програми Синхронізація даних і команди:
    • Настройки синхронізації даних - перехід до настройок і виконання синхронізації даних з іншими програмами (наприклад, з конфігурацій Бухгалтерія), а також організації спільної роботи в розподіленій інформаційній базі.
    • Дати заборони завантаження - увімкніть прапорець, для того щоб використати можливість заборони завантаження даних минулих періодів з інших програм. Див. також: Дати заборони зміни в підрозділі Загальні настройки.
      • Після цього стає доступною команда:
      • Настроїти - перехід до установки заборони завантаження даних минулих періодів з інших програм.
    • Попередження при синхронізації даних - перехід до перегляду попереджень, що виникли при синхронізації даних (непроведених документів, незаповнених реквізитів, конфліктів і пропущених за датою заборони даних).
    • Завантаження даних Enterprisedata - перехід до завантаження даних у форматі Enterprisedata без необхідності настройки регулярної синхронізації даних. Інформаційні бази повинні підтримувати синхронізацію даних через універсальний формат Enterprisedata. За допомогою цієї команди можливе лише завантаження даних;
    • Префікс цієї інформаційної бази - використовується для визначення програми, в якій були введені елементи списків і документи. До кодів елементів списків і номерів документів автоматично додається префікс інформаційної бази.

Каталог тимчасових файлів кластера серверів платформи

  • Шлях до єдиного мережевого каталогу для загальних тимчасових файлів серверів платформи, що працюють в одному кластері:
    • Під управлінням ОС Windows - вкажіть шлях до єдиного каталогу тимчасових файлів всіх серверів кластера, який буде використовуватися для серверів, що працюють під управлінням ОС Windows.
    • Під управлінням ОС Linux - вкажіть шлях до єдиного каталогу тимчасових файлів всіх серверів кластера, який буде використовуватися для серверів, що працюють під управлінням ОС Linux.
  • Каталоги тимчасових файлів можна не заповнювати, коли в кластері платформи працює тільки один сервер або програма працює у файловому режимі.
  • Якщо всі сервера кластера працюють під управлінням операційних систем одного сімейства (Windows/Linux), то друге поле заповнювати не обов'язково.
  • Каталог повинен бути доступний на читання і запис для всіх серверів кластера. В тому числі - для всіх користувачів операційних систем, від імені яких запускаються сервера платформи.
  • UNC шлях до каталогу повинен бути однаковим для всіх серверів одного сімейства - для цього каталог необхідно підключати до серверів під одним ім'ям.
  • У полі Каталог тимчасових файлів Linux можна вказати лише локальний шлях ,наприклад, / home / path.
  • Для того щоб використовувати UNC шлях, потрібно зробити на нього посилання. Для цього під правами адміністратора (root) зробіть наступне:
    • mkdir folder
      --- замість folder напишіть ім'я каталогу. наприклад, в каталозі / home робимо mkdir shared - ця команда створить каталог shared з повним шляхом / home / shared.
    • mount -t cifs //server/share folder -o //domain/user, password=pwd
      - де //server/share - UNC шлях до іншого комп'ютера , folder - шлях каталогу на комп'ютері Linux, наприклад \home\shared, //domain/user - ім'я домену/користувача, якщо потрібно, pwd - пароль користувача (якщо потрібно).
    • Якщо ім'я і пароль не потрібні, команда виглядає так:
      mount -t cifs //server/share folder
    • Після цього в дорозі до каталогу вказуємо folder, наприклад, /home/shared.
