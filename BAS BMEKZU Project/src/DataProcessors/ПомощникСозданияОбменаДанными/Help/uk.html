Призначений для настройки синхронізації даних. Можлива синхронізація даних між двома програмами за правилами обміну, без використання правил обміну або організація спільної роботи в розподіленій інформаційній базі (РІБ).

Вибір типу синхронізації даних

Виберіть потрібний розділ списку:

  • Обмін даних з іншими програмами;
    • Без використання правил обміну;
    • Повна синхронізація;
    • Отримання даних;
    • Повна синхронізація даних через EnterpriseData. При настройці обміну з використанням звичайних каналів зв'язку (файл, ftp, e-mail) в програмі-джерелі не вийде настроїти правила відправки та отримання даних, поки не будуть виконані настройки в програмі-приймачі.
    • Повна синхронізація з підтримкою зворотної сумісності;
    • Синхронізація даних через універсальний формат;
  • Розподілені інформаційні бази.

Етапи настройки параметрів синхронізації даних між двома програмами

  • Настройка синхронізації даних виконується в два етапи. Перший крок настройки синхронізації даних виконується в поточній програмі, потім необхідно перейти в іншу програму і закінчити в ній настройку синхронізації даних. Для передачі інформації між етапами настройки синхронізації даних використовується тимчасовий файл з настройками. Настройки правил надсилання і отримання даних виконуються в кожній з програм, що обмінюються, незалежно від обраного варіанту підключення.
  • Під час настройки визначаються наступні параметри:
    • Параметри підключення. Проводиться також перевірка працездатності введених параметрів;
    • Правила синхронізації даних;
    • Виконання початкового вивантаження даних;
    • У програмі-приймачі, крім перерахованих етапів, здійснюється також зіставлення і завантаження даних.

Особливості синхронізації через універсальний формат EnterpriseData

  • Настройку правил відправлення та отримання даних можна провести тільки після отримання параметрів програми-приймача.
  • Для цього після настройки варіантів підключення в програмі-джерелі необхідно перейти в програму-приймач, завершити в ній настройку і виконати первісне вивантаження даних, після цього завершити настройку в програмі-джерелі. Програма виводить про це повідомлення.

Вибір варіанту підключення до іншої програми

Перевірка варіанту підключення під час настройки проводиться програмою автоматично.

Для варіантів підключення, пов'язаних з Інтернетом, при необхідності з допомогою відповідної посилання настройте параметри проксі-сервера, перейшовши за посиланням Параметри доступу в Інтернет.

При вказівці користувача і пароля необхідно пам'ятати, що користувач повинен мати відповідний доступ (наприклад, до сервера платформи), для того щоб синхронізація даних відбулася.

За допомогою перемикача можна вибрати:

  • Пряме підключення на цьому комп'ютері або в локальній мережі використовується для синхронізації даних між двома програмами, причому обидві програми мають бути доступні для підключення.

    • Вкажіть параметри підключення до іншої програми:

      • Виберіть Розташування програми-приймача:
        • На даному комп'ютері або на комп'ютері в локальній мережі;
          • Виберіть Каталог інформаційної бази. Вкажіть шлях до папки на комп'ютері.
        • На сервері платформи;
          • Введіть Сервер;
          • Введіть Ім'я інформаційної бази.
      • за допомогою перемикача Параметри входу в програму виберіть:
        • Використовувати обліковий запис операційної системи;
        • Використовувати зазначені ім'я користувача та пароль:
          • Введіть ім'я Користувача, від імені якого проводитиметься синхронізація даних і його Пароль.
    • Натисніть кнопку Далі.
  • Підключення до програми через інтернет (веб-сервіс):
    • Такий варіант підключення використовується для синхронізації даних між двома програмами. Він може бути використаний у тому випадку, якщо програма-приймач знаходиться в іншому офісі, при цьому вона доступна для синхронізації через інтернет.
    • Введіть Адресу програми в інтернеті;
    • Введіть Ім'я користувача і його Пароль;
    • увімкніть прапорець зберегти пароль.
    • При необхідності настройте параметри проксі-сервера, перейшовши за посиланням Параметри доступу в Інтернет.
  • Синхронізація даних через файл, без підключення до іншої програми (мережевий каталог, FTP, електронна пошта);
    • Може використовуватися:
    • для синхронізації даних з програмою в іншому офісі, яка може працювати в автономному режимі, причому дві програми, які беруть участь у синхронізації даних, можуть працювати в різний час;
      • Для продовження настройки синхронізації в програмі-приймачі увімкніть прапорець Завантажити параметри підключення з файлу, вкажіть шлях до файлу з настройками.
    • Для синхронізації даних в розподіленій інформаційній базі.
  • Підключення для синхронізації даних виконується з іншої програми. При цьому варіанті підключення ця програма відправляє і приймає дані через Інтернет тільки за запитом від програми-кореспондента.

Натисніть кнопку Далі для переходу до подальшої настройки синхронізації даних.

Синхронізація даних через локальний або мережевий каталог

  • Увімкніть прапорець Використовувати локальний або мережевий каталог для синхронізації даних.

  • Для того щоб переконатися в правильності введених настройок, натисніть Перевірити можливість запису в каталог.

Синхронізація даних через каталог на FTP-сервері

  • Увімкніть прапорець Використовувати каталог на FTP-сервері для синхронізації даних.

  • При необхідності увімкніть прапорець Пасивний режим.
  • При необхідності перейдіть за посиланням Параметри доступу в інтернет і вкажіть параметри проксі-сервера.
  • Натисніть кнопку Далі для переходу до подальшої настройки синхронізації даних.

Синхронізація даних через електронну пошту

  • Увімкніть прапорець Використовувати електронну пошту для синхронізації даних.
  • В полі Обліковий запис виберіть зі списку або натисніть + Створити (клавіша F8) для додавання в список нового облікового запису.
  • Увімкніть прапорець Обмеження розміру вкладення, для того щоб обмежити розмір повідомлень, що відсилаються. Напишіть максимальний розмір вкладення.
  • Натисніть кнопку Далі для переходу до подальшої настройки синхронізації даних.

Додаткові настройки при синхронізації через файл

    Особливості синхронізації даних через універсальний формат або формат EnterpriseData через файл

    • При настройці синхронізації через формат EnterpriseData або через універсальний формат склад і інші характеристики одержуваних даних настроюються за допомогою правил відправки в програмі-кореспонденті.

    • Настройте варіант підключення через файл в програмі-джерелі;
    • Продовжите настройку синхронізації даних в програмі-приймачі. Настройте варіант підключення через файл.
    • Для цього перейдіть в програму-приймач, як варіанти підключення виберіть Синхронізація даних через файл, без підключення до іншої програми, вкажіть файл з настройками Синхронізація даних через універсальний формат.xml, отриманими з програми-джерела.
    • Тільки після проведення настройки варіантів підключення в обох програмах можна буде перейти до подальшої настройки синхронізації даних.

    Настройка правил відправлення та отримання даних

    Правила синхронізації даних використовуються, якщо необхідно синхронізувати не всі дані, а тільки частину.

    • За допомогою перемикача Документи можна визначити, як вони будуть синхронізуватися:
      • Відправляти і отримувати автоматично - документи будуть синхронізуватися в обох програмах;
      • Відправляти автоматично - документи будуть автоматично відправлятись в другу програму, у першій програмі автоматичного отримання документів з другої програми не буде;
      • Отримувати автоматично - документи будуть доставлятися з другої програми, документи відправлятися не будуть;
      • Відправляти і отримувати вручну - документи автоматично синхронізуватися не будуть. Відправку і отримання документів буде необхідно виконувати вручну.
    • За допомогою перемикача Нормативно-довідкова інформація виберіть режим вивантаження списків програми:
      • Відправляти і отримувати автоматично - дані будуть синхронізуватися в повному обсязі, нормативно-довідкова інформація буде автоматично синхронізуватися в цій і в другій програмі;
      • Відправляти і отримувати тільки використовувану в документах - дані будуть синхронізуватися в повному обсязі, нормативно-довідкова інформація буде автоматично синхронізуватися в цій і в другій програмі;
      • Відправляти і отримувати вручну - нормативно-довідкова інформація не буде автоматично синхронізуватися, її відправку і отримання необхідно буде виконувати вручну.
    • Вкажіть Дату початку відправки документів, по умовчанню значення дорівнює даті початку поточного року.
    • Можна обмежити дані, що беруть участь в синхронізації і задати, в розрізі яких Організацій, Складів і Підрозділів будуть відправлятися дані:
      • Включення фільтра За оранізаціями дозволяє обмежити список організацій, за якими дозволена синхронізація даних. Наявність включеного фільтра впливає як на відправку самого списку організацій, так і на відправку інших даних, пов'язаних з організаціями (списки і документи). Щоб увімкнути фільтр, установіть перемикач у вибране положення За обраними (організаціям), виберіть прапорці потрібні організації зі списку.
      • Для включення фільтра За підрозділами на відповідній закладці встановіть перемикач в положення За обраними (підрозділами), увімкніть відповідні прапорці.
      • Також можна вивантажувати інформацію За обраними (складами). Для цього на відповідній закладці за допомогою перемикача виберіть потрібне значення, за допомогою прапорців виберіть потрібні склади.
    • Правила отримання даних - значення по умовчанню необхідні для правильного відображення документів. Використовуються для автоматичного заповнення даних в документах, які не можуть бути отримані з обміну. Наприклад, в документах, отриманих з іншої програми, не були передбачені деякі реквізити. У цьому випадку при проведенні синхронізації даних ці реквізити будуть заповнені програмою автоматично. Їх можна вказати відразу ж, при настройці синхронізації (за посиланням Змінити значення автоматичного заповнення).

    Особливості настройки синхронізації даних в клієнт-серверному варіанті

    • При настройці синхронізації даних в клієнт-серверному варіанті роботи інформаційної бази необхідно мати на увазі, що дані синхронізуються на комп'ютері, на якому встановлено сервер платформи. Для сервера платформи під управлінням операційної системи Microsoft Windows це шлях виду:
      \\servername\геѕоигсе
      а для сервера під управлінням Linux:
      //servername/resource.
    • Необхідно переконатися в тому, що каталог синхронізації доступний для читання і запису користувачеві операційної системи, від імені якого виконується сервер платформи. Якщо це не так, то необхідно дати цьому користувачу необхідні права, або вибрати іншого користувача для запуску сервера платформи з необхідними правами на каталог, який використовується для синхронізації даних. Ці ж рекомендації актуальні і для доступу до ресурсів FTP і електронної пошти. Докладніше про адміністрування сервера платформи див. документацію до платформи.

    Особливості створення вузла РІБ.

    На першому етапі створюється початковий образ РІБ. Всі сторінки помічника залишаються без змін. На сторінці Перший етап настройки успішно завершений по умовчанню включений прапорець Створити початковий образ підлеглого вузла РІБ (рекомендуємо це зробити відразу). Цей крок можна зробити і пізніше, з настройки синхронізації в будь-який момент.

    • Після натискання на кнопку Готово з включеним параметром пропонується створити початковий образ РІБ. Для цього перейдіть за посиланням Створити початковий образ.
    • По умовчанню файл з настройками для створення вузла РІБ буде створений в каталозі, який був обраний для обміну, На даному комп'ютері або на комп'ютері в локальній мережі.
    • Однак можна за допомогою перемикача вибрати створення На сервері платформи. У цьому випадку необхідно заповнити Параметри інформаційної бази:
    • Введіть Кластер серверів платформи.
    • Напишіть Ім'я інформаційної бази в кластері.
    • Виберіть параметри в поле Захищене з'єднання.
    • Виберіть Тип СУБД.
    • Вкажіть Сервер бази даних.
    • Напишіть Ім'я бази даних і
    • Заповніть поле Користувач бази даних, введіть Пароль користувача.
    • Вкажіть Зміщення дат.
    • Вкажіть Мову програми.
    • За необхідності увімкніть прапорець Встановити блокування регламентних завдань.
    • Натисніть Створити початковий образ.
    • Створений файл з початковим чином підпорядкованого вузла розподіленої інформаційної бази збережіть на комп'ютері філії в нову папку, яка буде робочим каталогом нової бази.
    • Створіть нову базу, вибравши Додавання в список існуючої інформаційної бази, вкажіть шлях до файлу з початковим чином. Зазначена інформаційна база буде додана в список баз.
    • Після входу в програму автоматично запуститься Помічник на сторінці Продовження настройки синхронізації даних з головним вузлом, продовжте настройку синхронізації, натисніть кнопку Далі, щоб продовжити настройку і задати варіант підключення до головного вузла.

    Створення вузла РІБ (зовнішні файли)

    • Якщо у вас є файли у томах, то при створенні початкового образу вам також потрібно буде вказати шлях до файла, в який буде записаний архів (zip) з усіма файлами в томах. При першому запуску в базі філії вам потрібно буде виконати команду Розмістити файли початкового образу, вказавши в ній шлях до файлу архіву.
